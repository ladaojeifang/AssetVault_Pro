## Eagle 资源包（Eaglepack）完整解析

### 一、什么是 Eaglepack？

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   📦 Eaglepack = Eagle 专属素材封装格式 (.eaglepack)        │
│                                                             │
│   类似于一个"设计素材的压缩包"                               │
│   但比普通压缩包保留了更多元数据信息                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

| 属性 | 说明 |
|------|------|
| **文件格式** | `.eaglepack` |
| **本质** | Eagle 软件专属的素材封装格式 |
| **用途** | 素材分享、备份、迁移 |
| **唯一性** | 只能在 Eagle 软件中打开 |

---

### 二、Eaglepack vs 普通导出

| 对比项 | Eaglepack 导出 | 普通导出 |
|--------|---------------|----------|
| **文件格式** | `.eaglepack` | 原始文件（图片/视频） |
| **元数据保留** | ✅ 完整保留 | ❌ 仅文件本身 |
| **标签保留** | ✅ 保留 | ❌ 不保留 |
| **文件夹结构** | ✅ 保留 | ❌ 不保留 |
| **评分保留** | ✅ 保留 | ❌ 不保留 |
| **注释保留** | ✅ 保留 | ❌ 不保留 |
| **来源网址保留** | ✅ 保留 | ❌ 不保留 |
| **适用场景** | 团队分享、备份 | 单纯导出文件 |

### 六、注意事项

| 事项 | 说明 |
|------|------|
| **标签群组** | 导出时不包含标签群组结构 |
| **跨设备导入** | 保留完整文件夹层级结构 |
| **文件依赖** | 包内包含原始文件，无需外部依赖 |
| **大小限制** | 取决于打包的素材数量和大小 |

### 八、总结

```
Eaglepack 核心价值：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   🎁 一包一分享 = 文件 + 结构 + 信息 + 标注                  │
│                                                             │
│   不是简单的"文件压缩"，而是"素材知识库打包"                 │
│                                                             │
│   • 保留创作者的整理思路                                     │
│   • 保留素材的分类逻辑                                       │

│                                                             │
```

**一句话总结**：Eaglepack 是 Eagle 的"素材胶囊"，将文件本身与整理信息（标签、分类、评分、注释）封装在一起，实现完整的素材知识传递。
