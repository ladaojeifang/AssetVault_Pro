产品概述

1.1 产品定位

AssetVault Pro 是一款面向创意设计师、设计团队和企业的智能数字资产管理（DAM）桌面应用软件。它致力于解决设计师在日常工作场景中面临的素材管理痛点——海量图片、字体、视频、3D模型等资产的收集、整理、检索与协作效率低下的问题。


本产品以「极简、极速、智能」为核心理念，通过先进的本地优先架构、AI驱动的智能分类系统、以及卓越的大数据量性能优化能力，帮助用户实现资产管理的数字化升级。

1.2 产品愿景

  • 成为全球领先的本地化数字资产管理解决方案
  • 让每一位创作者都能高效管理和复用其数字资产
  • 构建开放的插件生态系统，连接全球创意工具链
  • 通过AI技术降低资产管理的人工成本90%以上

1.3 核心价值主张

价值维度具体描述
性能极致支持单目录10万+文件流畅浏览，导入速度提升300%，内存占用优化至竞品1/5
隐私安全100%本地存储，端到端加密，用户完全掌控自己的数据主权
AI赋能自动标签识别、相似图搜索、智能推荐、OCR文字提取，大幅降低人工分类成本
生态开放丰富的插件API、浏览器扩展、Adobe/Figma集成，无缝融入现有工作流
跨平台Windows / macOS 双平台原生体验，数据格式互通，团队协作无障碍

2.2 关键使用场景

🔍灵感收集

浏览网页时一键收藏图片/配色/字体到本地库。通过浏览器扩展捕获，自动提取元数据和来源URL，智能归档到对应文件夹。

💾批量导入

一次性导入5000+张项目素材图片。后台异步处理缩略图生成、颜色提取、标签识别，UI保持流畅响应不卡顿。

🔍︎快速检索

用颜色/形状/关键词在海量库中秒级找到目标素材。多维度组合筛选 + AI语义搜索 + 以图搜图，支持模糊匹配和相似度排序。

⚙️工作流嵌入

从素材库直接拖拽到PS/AI/Figma中使用。原生插件集成，支持拖放操作、格式自动转换、智能图层命名。

👥团队协作

多人同时访问共享素材库，实时同步更新。基于Git-like的增量同步机制，冲突检测与合并，离线支持与断点续传。

3.2 M01 — 资产管理引擎（P0）

📁 导入系统

  1. 多源导入：拖拽文件夹、剪贴板粘贴、浏览器扩展一键保存
  2. 批量处理：单次支持导入50,000+文件，实时进度 + 可取消/暂停
  3. 格式支持：JPG/PNG/WebP/SVG/GIF/HEIC/AVIF/PSD/AI/TIFF/raw/MP4/MOV/字体/3D模型
  4. 智能去重：感知哈希(pHash)算法检测重复图片
  5. 元数据提取：自动读取EXIF/IPTC/XMP信息

🗂️ 组织系统

  1. 嵌套文件夹：无限层级结构，拖拽排序、批量移动
  2. 智能文件夹：基于规则的动态集合（最近7天/红色系/未打标签）
  3. 标签体系：多级层级结构、颜色、别名、标签云视图
  4. 收藏夹：全局收藏，置顶常用文件夹和资产
  5. 颜色分组：自动提取主色调，按色相环分组

🖼️ 三级预览管线

  • L1 缩略图 — 128×128px WebP格式，质量60%，目标<8KB/张，用于网格视图
  • L2 预览图 — 最长边1920px JPEG格式，质量85%，目标<300KB/张，用于详情/全屏
  • L3 原始文件 — 按需加载，双击或进入全屏预览时触发
  • 全屏预览模式：幻灯片播放 / 键盘导航 / EXIF侧边栏 / 缩放平移
  • 视频内置播放器 + GIF动画 + 音频波形 + 字体实时渲染 + 3D WebGL预览

4.3 数据库 Schema（核心表）

SQLite Schema
-- ════════════════════════════════════════════
--  资产主表 assets
-- ════════════════════════════════════════════
CREATE TABLE assets (
  id             INTEGER PRIMARY KEY AUTOINCREMENT,
  name           TEXT NOT NULL,
  original_path  TEXT NOT NULL UNIQUE,
  file_size      INTEGER NOT NULL,
  file_type      TEXT NOT NULL,         -- image/video/font/document/3d/audio
  mime_type      TEXT,
  width          INTEGER,
  height         INTEGER,
  duration       REAL,
  color_palette  TEXT,                  -- JSON数组 主色调hex值
  thumbnail_hash TEXT UNIQUE,
  preview_hash   TEXT UNIQUE,
  phash          BLOB,                  -- 感知哈希 用于去重
  embedding      BLOB,                  -- AI向量 用于以图搜图
  metadata       TEXT,                  -- JSON EXIF/IPTC/XMP
  is_deleted     INTEGER DEFAULT 0,
  created_at     TEXT DEFAULT (datetime('now')),
  updated_at     TEXT DEFAULT (datetime('now'))
);

-- ════════════════════════════════════════════
--  标签表 tags (支持多级层级)
-- ════════════════════════════════════════════
CREATE TABLE tags (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  name        TEXT NOT NULL UNIQUE,
  color       TEXT DEFAULT '#888888',
  parent_id   INTEGER REFERENCES tags(id),
  usage_count INTEGER DEFAULT 0
);

-- 资产-标签关联表 (多对多)
CREATE TABLE asset_tags (
  asset_id  INTEGER REFERENCES assets(id) ON DELETE CASCADE,
  tag_id    INTEGER REFERENCES tags(id) ON DELETE CASCADE,
  PRIMARY KEY (asset_id, tag_id)
);

-- ════════════════════════════════════════════
--  文件夹表 folders (含智能文件夹)
-- ════════════════════════════════════════════
CREATE TABLE folders (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  name        TEXT NOT NULL,
  parent_id   INTEGER REFERENCES folders(id),
  sort_order  INTEGER DEFAULT 0,
  is_smart    INTEGER DEFAULT 0,
  smart_rules TEXT,                    -- JSON 规则表达式
  created_at  TEXT DEFAULT (datetime('now'))
);

-- ════════════════════════════════════════════
--  全文检索虚拟表 (FTS5)
-- ════════════════════════════════════════════
CREATE VIRTUAL TABLE assets_fts USING fts5(
  name, description, tags,
  content = assets,
  content_rowid = id
);

-- ════════════════════════════════════════════
--  性能关键索引
-- ════════════════════════════════════════════
CREATE INDEX idx_assets_folder  ON assets(folder_id);
CREATE INDEX idx_assets_type     ON assets(file_type);
CREATE INDEX idx_assets_created  ON assets(created_at DESC);
CREATE INDEX idx_assets_color    ON assets(color_palette);

性能优化策略（核心技术壁垒）

性能是数字资产管理产品的生命线。以下策略确保在10万级资产库中依然保持丝滑流畅。

5.1 虚拟滚动引擎

  • 原理：只渲染可视区域 + 缓冲区内的DOM节点（约25-35个），而非全部资产项
  • 网格布局：自适应列数计算，响应窗口resize动态重排
  • DOM对象池：复用已滚出视口的DOM节点，避免频繁创建/销毁
  • GPU加速：使用transform: translateY()代替top属性触发硬件加速层
  • 滚动节流：requestAnimationFrame节流至60fps

5.5 懒加载与预测性预取

  • IntersectionObserver：rootMargin='500px' 提前触发可视区外500px元素加载
  • LQIP占位策略：先加载极低质量缩略图(~1KB)做模糊背景，高清图到位后淡入替换
  • 滚动方向感知：向下滚动时预取下方内容，向上时反转方向
  • 鼠标轨迹预测：分析鼠标移动向量，预取可能点击区域的预览图

8

非功能性需求

✅ 兼容性要求

  • Windows: 10 (21H2+) / 11 x64
  • macOS: 12+ Intel + Apple Silicon Universal Binary
  • 最低内存: 8GB RAM (推荐16GB+)
  • 磁盘: 应用~150MB + 可配置缓存上限
  • 显示器: 最低1280×720, 推荐1920×1080+, HiDPI支持

🔒 安全性与隐私

  • 本地优先: 所有数据默认存储在本地
  • 加密存储: 敏感配置使用系统密钥链
  • 代码签名: Authenticode + Apple Notarization
  • 最小权限: 仅请求必要的系统权限
  • 合规: 符合GDPR/个人信息保护法

♿ 可访问性(A11y)

  • 屏幕阅读器支持: ARIA标签完整覆盖
  • 键盘完全可操作: Tab序列合理无陷阱
  • 色彩对比度: WCAG AA标准 (≥4.5:1)
  • DPI缩放: 支持125%/150%/200%

📋 可维护性

  • 日志系统: 结构化JSON, DEBUG/INFO/WARN/ERROR, 自动轮转
  • 错误追踪: Sentry自托管, 崩溃报告自动收集
  • 数据备份: 手动/定时备份, 增量备份
  • 数据迁移: 旧版本兼容升级路径
